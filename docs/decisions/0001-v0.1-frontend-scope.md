# ADR 0001: v0.1 Frontend Scope

**Status:** Accepted

**Context:** The repository needed a clear scope boundary for its first
implementation stage. Without a defined scope, contributors would be tempted
to implement semantic passes prematurely.

**Decision:** v0.1 is strictly `source text -> tokens -> AST -> diagnostics`.
No type checking, kind checking, overload resolution, canonical matching,
closure materialization, match/effect/sync semantics, ownership/NLL/drop
analysis, interpretation, or code generation.

**Consequences:**
- Parser must preserve syntax sufficient for future semantic passes.
- Semantic names (`return`, `else`, `match`, etc.) are ordinary `Name` tokens.
- Closure literals produce `ClosureAST`, not `ClosureObject`.
- `xtask` is optional tooling, not part of v0.1 semantics.
