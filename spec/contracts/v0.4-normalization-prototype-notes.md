# v0.4 Normalization Prototype Notes

v0.4 starts with a broad, shallow normalized lowering loop:

```text
Raw AST -> Normalized AST -> stable dump -> golden tests
```

The first implementation pass prioritizes structural coverage and
observability over full lowering completeness. Unsupported Raw AST subshapes
remain visible in the normalized dump instead of being silently erased.

Value-side `NormExpr` and pattern-side `NormPattern` remain distinct. Raw
expression-shaped syntax may be normalized as pattern material only when the
surrounding syntactic context is pattern, annotation, or extraction. Value to
pattern conversion requires an explicit bridge; pattern to value requires
extraction. This prototype records that boundary; explicit bridge
syntax/lowering is future work unless it is already present in Raw AST.

The crate `VERSION` remains tied to the frozen v0.2 public frontend surface in
this prototype. This PR adds v0.4 normalization scaffolding without changing
the public lexer/parser/raw-AST contract version.

This prototype remains non-semantic. It does not perform name resolution,
type checking, kind checking, operator lookup, alias target resolution,
closure materialization, capture analysis, ownership analysis, runtime
evaluation, or code generation.
